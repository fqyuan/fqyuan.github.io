Markdown使用介绍

1. 为什么使用markdown

MarkDown是由John
Gruber，一个UI设计师兼作家在2004年发明的。他创立此种格式的目的是为了让人们在创立HTML网页时更专注于写作过程，而不需要过于关注HTML语法的tags等格式问题。Markdown并不是用来取代HTML(hypertext
markup language),
它主要为了补充增强html的能力–它是一种写作语言，而html是一种发布语言。

理想的目标：找到一种适合打印同时也适合在浏览器发布的格式、预览方便、允许很精细的内容管理特征并支持复用、和源代码管理工具很好集成这几种特点。

1.1 Markdown优点

  • 简单易用：规则简单，可以很方便编写。
  • 预览方便：可以从源文件获取绝大部分信息，而不必过于关注额外的格式信息。
  • 版本管理友好：由于md格式文件基本都是基于文本，所以可以很方便比较版本间差异。

1.2 Markdown缺点

  • 不适合排版：如居中，左右对齐、字体选择等所谓的排版格式。
  • 不适合较长的内容：如果你管理超过一页内容的text文档，我们不建议使用markdown。
  • 扩展性差：MarkDown设计初衷是简单易用，很难扩展更多复杂的特性。

2. 使用什么工具编写

2.1 Visual Studio Code

开源编辑器，建议搭配MarkDown插件使用。

2.2 Typora

开源的markdown编辑器，提供了一种编辑和预览之间的无缝体验，推荐。搭配Pandoc使用可以方便进行文件格式转换。

3. 生成目录方式

3.1 typora

3.1.1 在文档顶部首行加上TOC标记

文档顶部首行增加[TOC]即可，TOC–Table of contents简写。

3.1.2 配合pandoc使用

-s：–standalone，表示不输出header和footer信息

–toc: 标识生成目录接口

–toc-depth:默认为3，可以自己指定

original_file.md: 待更新目录文件

-o : 输出，output

file_with_toc.md：带有目录的新文件

3.2 visual studio code

安装MarkDown TOC插件，在对应位置Ctrl + Shift + P, 输入Generate即可生成。
